# Task 2: Use Case Diagrams

This document provides a brief description of the diagrams included in the `task2.pdf` file. All diagrams in this document are Use Case Diagrams representing the interactions between different actors and the system.

## 1. ATM (Automated Teller Machine) - Use Case Diagram
This diagram illustrates the interactions between a **Customer**, the **ATM**, and the **Bank server**. 
- **Key Use Cases:** Deposit, Withdraw, Change PIN, and Check account balance.
- **Includes/Extends:** Deposit and Withdraw actions include the choice of using 'Blik' or 'Card'. Withdrawing money can optionally extend to 'Print receipt'. 

## 2. Car Insurance System - Use Case Diagram
This diagram outlines the operations a **Customer** and an **Insurance agent** can perform within a car insurance system.
- **Key Use Cases:** Pay for insurance, File a claim, View history of claims, Accept/Reject a claim, and Withdraw money for customer.
- **Includes/Extends:** Paying for insurance includes options for 'Cash' or 'Card'. Filing a claim includes the agent's action to Accept or Reject it.

## 3. Medical Clinic System - Use Case Diagram
This diagram shows the interactions between a **Patient** and a **Doctor** within a medical clinic system.
- **Key Use Cases for Patient:** Book Appointment, Cancel Appointment.
- **Key Use Cases for Doctor:** Check medical history, Prescribe drugs, and Prescribe a treatment.
